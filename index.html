關於我

我是張誌紘，畢業於國立台灣大學圖書資訊學系。自大二開始，在課程中稍微接觸了程式語言，因感到很有趣而開始自學程式。部分程式作品有放在我的 GitHub 上，雖然沒有太多實務上的經驗，但相信仍能夠完成被指派的任務。

我的興趣相當廣泛，平日喜歡聽音樂和讀小說，也喜歡打桌球和羽毛球。此外，從小就喜歡《精靈寶可夢》系列（舊譯《神奇寶貝》），並以此為契機，在大學期間自學日文。

網站作品

捷運好食在

在「數位圖書館」課程中，使用線上數位典藏內容管理系統 Omeka 所建置。

在蒐集資料前，先參考國內外菜單收藏的相關網站，並以此為基礎，依據都柏林核心集（Dublin Core）的內容設計著錄規範。最後，挑選流量前五十名的台北捷運站附近的代表性餐廳，蒐集其菜單後進行數位化、組織整理、典藏保存並提供閱覽使用。

負責項目

  • 菜單資料數位化
  • 著錄規範設計
  • 網站後台管理

偵探書屋

在「資訊結構與網站設計」課程中，先透過標竿學習法評估國內外各獨立書店網站的優缺點。其後，透過與台北市內的獨立書店「偵探書屋」的訪談，規劃設計出適合這間書店的網站。

網站 protoype 為使用 Bootstrap 3 前端框架建置的靜態網站。

負責項目

  • 頁面建置及整體管理

移送書編輯系統

在「系統分析與設計」課程中，與內政部警政署刑事警察局合作，將原先使用者反映使用上有些問題的移送書編輯系統網站化，期望協助並改善外勤員警編輯移送書之流程，建立更符合各方需求的系統。

此系統 prototype 前端以 JQuery 達成和使用者的互動，後端以 Node.js 及 MongoDB 控制資料流動。

負責項目

  • use case diagram 繪製
  • 前端頁面初步設計
  • 表單前端驗證

圖資系實習圖書館

在「圖書資訊機構實務一 」課程中，用網站做為圖書館導覽與資源利用教育的媒介。

圖資系上的官方網站為利用 WordPress 建置的動態網站，而此網站為靜態網站，僅針對原先內容貧乏的「實習圖書館」頁面翻新內容，並模仿官網的版型呈現，後於課程結束後提供給系上參考。

負責項目

  • 網站整體建置

京都的留學日誌

在「網際網路程式設計」課程中，嘗試建立了留言板系統。日後以當時的成品為基礎，擴展為部落格系統，並被用來記錄在日本交換留學時的生活與心得。

系統前端為網路上開源的 template，僅依個人所需些微調整。後端以 Java 的 JSP / Servlet ，以及 MySQL 建置，並用 AWS Elastic Beanstalk 的 PaaS 服務進行託管。目前改以 Red Hat 的 Openshift 做託管。

負責項目

  • 網站前端模板調整
  • 網站後端建置
